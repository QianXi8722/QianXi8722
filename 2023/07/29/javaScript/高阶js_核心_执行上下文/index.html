执行上下文可以理解为当前代码的运行环境

执行上下文可以理解为当前代码的运行环境

  • JavaScript中的运行环境主要包括以下三种情况。
  1. 全局环境： 代码运行起来后会首先进入全局环境。
  2. 函数环境： 当函数被调用执行时，会进入当前函数中执行代码。
  3. eval环境：不建议使用，这里不做介绍

一个JavaScript中，必定会出现多个执行上下文。JavaScript会以栈的方式来处理他们，这个栈就是函数调用栈。函数调用栈规定了JavaScript代码的执行顺序。栈底永远都是全局上下文，栈顶则是当前正在执行的上下文。
以上几种情况都会生成一个执行上下文并放入函数调用栈中，处于栈顶的上下文执行完毕后，会自动出栈。

// demo2.js
function f1() {
 var n = 999;
 function f2() {
  alert(n);
 }
 return f2; // 只写f2不代表执行   f2()才是执行
}
var result = f1();
result(); // 999
  • 这是一个简单闭包例子
    1. 第一步，仍然是全局上下文先【入栈】 【Global Context】
    1. 全局代码在执行过程中，遇到f1()函数，执行 var result = f1(); f1创建对应的执行上下文并【入栈】
    1. 在f1的可执行代码中，虽然声明了一个函数f2，因为这个时候会执行f2中的代码。由于f2中没有产生新的上下文，因此执行完毕后直接【出栈】。
    1. f1出栈后，继续执行全局上下文的代码，这个时候遇到了result()即f2, result()会创建一个新的上下文，因此这个时候result的上下文【入栈】
    1. 这个result()其实就是在f1中声明的函数f2，因此这个时候会执行f2中的代码。由于在f2中没有产生新的上下文，因此执行完毕后直接【出栈】

生命周期

当一个函数调用时，一个新的执行上下文就会被创建。一个执行上下文的生命周期大致可以分为两个阶段：创建阶段 和 执行阶段

创建阶段
在这个结算，执行上下文会分别创建变量对象，确认作用域链，以及确定this的指向
执行阶段
创建阶段之后就开始执行代码，这个时候会完成变量赋值、函数引用。以及执行其它可执行代码
执行上下文生命周期
生成变量对象 变量赋值
创建—>确认作用域链 ———— 执行—> 函数引用 —————— 执行完毕后出栈，等待被回收
确定this指向 执行其它代码
