javaScript有垃圾回收机制

javaScript有垃圾回收机制

1 基础数据类型与变量对象

最新的ECMAScript标准号定义了7中数据类型，其中包括六种基础数据类型与一种引用数据类型

  • 类型 值
  • Boolean 只有两个值true false
  • Null 只有一个值：null
  • Undefined 只有一个值:undefined
  • Number 所有的数字
  • String 所有的字符串
  • Symbol 符号类型 var sym = Symbol(‘testsymbol’) 【常用浏览器不支持，暂时不要用】

引用数据类型与堆内存空间

  • 【基础数据类型】数据发生一次复制行为，复制的是【值】而不是 指针地址
  • 【引用数据类型】数据发生一次复制行为，实际上复制的是【指针地址】
  • 当地址指针相同时，尽管堆内存空间中得变量相互独立，但他们指向的具体对象实际上是同一个

内存空间管理

  • 简单的例子了解内存空间的使用过程
    // 1. 分配内存
var a = 20;
// 2. 使用分配到的内存
alert(a + 100);
// 3. 不需要时释放内存
a = null;

    第三个过程涉及JavaScript垃圾回收机制的实现原理。JavaScript的垃圾回收实现主要依靠“引用”的概念。当一块内存空间中的数据能够被访问时，垃圾回收器就认为“该数据能能够被获得”。不能够被获得的数据，就会被打上标记，并回收内部才能空间。这种方式叫做【标记-清除算法】

注意：局部作用域中，当函数执行完毕后，局部变量就没必要存在了，因此垃圾收集器很容易判断并回收，但在全局中，全局变量什么时候需要自动释放内存很难判断，因此开发时尽量避免使用全局变量
