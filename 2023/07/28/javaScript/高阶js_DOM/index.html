DOM是javaScript操作网页的接口，全程为“文档对象模型”。他的作用是将网页转为一个javaScript对象，从而可以用脚本进行各种操作（比如增删内容）

DOM是javaScript操作网页的接口，全程为“文档对象模型”。他的作用是将网页转为一个javaScript对象，从而可以用脚本进行各种操作（比如增删内容）

Document节点

document.location Location对象是浏览器提供的原生对象，提供URL相关的信息和操作方法。通过window.location的属性，可以拿到这个对象

EventTarget 接口

addEventListener: 绑定事件的监听函数
removeEventListener: 移除事件的监听函数
dispatchEvent: 触发事件 是否

function hello() {
  console.log('hello world!');
}
var button = document.getElementById('btn');

/**
 * 第一个参数： 事件名称  
 * 第二个参数：事件处理成语function    
 * 第三个参数：boolean值  是否以捕获事件机制执行 若不指定则预设为冒泡
 **/
button.addEventListener('click', hello, false)

事件的传播

一个事件发生后，会在子元素和父元素之间传播（propagetion）。这种传播分为三个阶段

  • 第一阶段： 从window对象传导到目标节点（上层传到底层），成为”捕获阶段”
  • 第二阶段： 在目标节点上触发，称为”目标阶段”
  • 第三阶段： 从目标节点传导回window对象（从底层传回上层）,称为”冒泡阶段”

事件的代理

由于事件会在冒泡阶段向上传播到父节点，因此可以把子节点的监听函数定义在父节点上，由父节点的监听函数统一处理多个子元素的事件。这种方法叫做事件的代理

var url = document.querySelector('ul');
